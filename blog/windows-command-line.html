Part 2: Navigating the File System with CMD & PowerShell

First, open the Command Prompt or PowerShell:

  • Command Prompt (cmd.exe): Press Win + R, type cmd, and hit Enter.
  • PowerShell: Press Win + X and select Windows PowerShell.

1. See Your Current Location

CMD:


echo %CD%

PowerShell:


pwd  # Returns the current directory
$PWD  # PowerShell object way, same result as `pwd`

When to use: In CMD, echo %CD% prints the current directory, and in PowerShell, pwd gives the current directory as well. The PowerShell object way, $PWD, works well in scripts and when using PowerShell interactively, as it's more flexible for grabbing current directory values programmatically. Both methods are useful for confirming where you are in the file system before performing file operations.

2. List Files and Folders

CMD:


dir

PowerShell:


ls

When to use: Both commands will list files in the current directory. PowerShell uses ls as an alias for the UNIX-style ls command, which is a little more user-friendly for those familiar with Linux or macOS. CMD uses dir, which has additional parameters to control the output. For example, use dir /w to list files in a wide format in CMD. In PowerShell, ls can also accept additional filters and more advanced options, giving you more flexibility when handling file listings.

3. Move Around (Change Directory)

CMD:


cd C:\Users\John\Documents  # Move to Documents folder
cd ..                       # Move up one level
cd /d D:\Projects           # Switch to another drive

PowerShell:


cd C:\Users\John\Documents  # Move to Documents folder
cd ..                       # Move up one level
cd /d D:\Projects           # Switch to another drive (works like CMD)
Set-Location C:\Users\John  # Alternative PowerShell cmdlet to change directory

When to use: In CMD, you use cd to move between directories, and to switch drives, you add /d. PowerShell supports cd as an alias, but it also offers Set-Location for clearer intentions, especially in scripts. When you're working interactively or scripting in PowerShell, Set-Location is more precise when you need to manipulate paths or file system objects programmatically.

4. Create New Folders

CMD:


mkdir MyFolder

PowerShell:


mkdir MyFolder
New-Item MyFolder -ItemType Directory  # Alternative in PowerShell

When to use: The mkdir command works the same in both CMD and PowerShell for creating directories. However, PowerShell's New-Item allows you to specify the type of item being created (file or directory), which is helpful if you want more control over the process in a script.

5. Create Files

PowerShell:


New-Item myfile.txt -ItemType File

CMD:


echo "Hello World" > myfile.txt

When to use: PowerShell offers New-Item to create files with specific types, while CMD simply uses redirection with echo to create files. The PowerShell approach is better when you need to create files in more complex workflows, while CMD is faster for quick file creation.

6. Delete Files and Folders

CMD:


del myfile.txt
rmdir MyFolder

PowerShell:


Remove-Item myfile.txt
Remove-Item MyFolder -Recurse

When to use: CMD uses del for deleting files and rmdir for removing directories. PowerShell combines these with Remove-Item, which can delete both files and directories, with the -Recurse flag for removing folder contents. PowerShell‚Äôs method is more versatile and recommended for complex file management.

7. Copy and Move Files

CMD:


copy myfile.txt C:\Backup\

PowerShell:


Move-Item myfile.txt C:\Backup\
Copy-Item myfile.txt C:\Backup\

When to use: CMD‚Äôs copy and PowerShell‚Äôs Copy-Item are similar, but PowerShell‚Äôs Move-Item is more intuitive for moving files compared to CMD‚Äôs move. PowerShell's cmdlets offer additional parameters like -Force or -Recurse to deal with read-only files and folder contents.

Key File Locations & Tools Available in the CLI

Now that you've learned how to navigate and manage files in the command line, it's important to understand where to find key system files and the tools you have access to in the CLI. Here's a breakdown of the most commonly used directories and utilities that will be helpful when you're working with files in Windows:

Key File Locations

Here are some important file locations in Windows that you will frequently interact with through the command line:

  • C:\ - The root directory of the system. This is where your Windows installation resides, along with important folders like Program Files, Windows, and Users.
  • C:\Users\YourName\ - Your personal user directory. This is where your documents, desktop files, and settings are stored. It's the equivalent of your "home" directory in other operating systems like Linux or macOS.
  • C:\Program Files\ - Contains installed programs (32-bit applications are usually stored here, while 64-bit apps are placed in C:\Program Files (x86)\).
  • C:\Windows\ - The system directory containing all the operating system files. This is a crucial folder and shouldn't be modified unless you're experienced.
  • C:\Temp\ - A folder that contains temporary files. You may want to clean this up periodically to free space.
  • C:\ProgramData\ - Stores application data that is shared between users. This folder is hidden by default, but it's important for managing software and settings in Windows.
  • C:\Users\YourName\AppData\ - A hidden folder where applications store user-specific data. This includes settings, cache, and other essential data files.

Note: You can navigate directly to these locations using the cd command in the CLI. For example, to get to your personal folder, you can run cd C:\Users\YourName\.

Essential Command Line Tools

Here are some useful tools and utilities available in the Windows command line that you'll often interact with:

  • dir - Lists the contents of a directory. You can also use dir /s to search for files within subdirectories or dir /b to list filenames without additional details.
  • cd - Changes the current directory. This is the most used navigation command. You can use it to change to any folder by specifying its path (e.g., cd C:\Users\YourName\Documents).
  • mkdir - Creates a new directory. For example, mkdir MyFolder creates a folder named "MyFolder" in the current directory.
  • del - Deletes a file. You can also use del /f to force deletion of read-only files or del /s to delete files in subdirectories.
  • copy - Copies files from one location to another. For example, copy file1.txt C:\Backup\ copies file1.txt to the C:\Backup folder.
  • move - Moves files from one location to another. This is used when you want to change the location of a file.
  • xcopy - A more powerful version of copy that allows for copying entire directories, including subdirectories.
  • robocopy - Similar to xcopy but more advanced. It's ideal for copying large amounts of data or dealing with complicated directory structures. It also supports resuming broken copies and copying over network locations.
  • findstr - Searches for text within files. It‚Äôs similar to grep in Linux. You can use it to find occurrences of a specific string in one or more files.
  • choco - The Chocolatey package manager command. Use this tool to install, update, or manage software directly from the command line (e.g., choco install git).
  • wmic - A powerful tool for accessing system information, including hardware, software, and system configuration. For example, wmic logicaldisk get size,freespace,caption shows the disk usage of all drives.
  • tasklist - Displays a list of all running processes. This is useful for monitoring system activity.
  • taskkill - Terminates a running process. You can use this to end any process by specifying its name or PID (process ID).

Note: Most of these tools work in both CMD and PowerShell, but PowerShell commands might be more advanced and offer additional functionality. If you're working in PowerShell, you may encounter cmdlets (e.g., Get-Process) that are specific to PowerShell, and not always available in CMD.

File Path Environment Variables

In Windows, certain environment variables help you access commonly used directories without typing out the full path. Here are some examples:

  • %USERPROFILE% - Refers to C:\Users\YourName\, your personal user directory.
  • %APPDATA% - Refers to C:\Users\YourName\AppData\Roaming\, where application settings are stored.
  • %TEMP% - Refers to your temporary files directory (C:\Users\YourName\AppData\Local\Temp).
  • %PROGRAMFILES% - Refers to the C:\Program Files directory, where installed programs are stored.
  • %WINDIR% - Refers to the C:\Windows directory, where Windows system files are stored.

Using these environment variables can save you time when navigating through the system or referencing directories in commands. You can quickly access the system's temporary folder with cd %TEMP%, for example.
